Imperative VS Declarative

Grunt (Declarative)

concat: {
  options: {
    separator: ';'
  },
  dist: {
    src: ['./lib/**/*.js'],
    dest: 'dist/all.js'
  }
}

Gulp (Imperative)

gulp.src("./lib/**/*.js")
  .pipe(concat("all.js", {newLine: ";"}))
  .pipe(gulp.dest("./dist/"));

Using gulp

Four Things

  • gulp.src
  • gulp.dest
  • gulp.task
  • gulp.watch

gulp.src

  • Takes a list of globs
  • Returns a stream
  • Preserves order of files in list

gulp.dest

  • Takes a path to a destination directory
  • Returns a stream

gulp.task

  • Is exposed to the command line
  • Interfaces with gulp's dependency management
  • Asynchronous by default

gulp.watch

  • Takes a list of globs
  • Triggers a task or function

Pipes

Three ways

Tasks are async by default.
Signaling is up to you!

  1. Return a stream
  2. Callback
  3. Return a promise

1. Return a Stream

gulp.task('jade', function() {
  return gulp.src(sources.jade)
  .pipe(jade())
  .pipe(gulp.dest("./public/"));
});

2. Callback

gulp.task('sass', function(cb) {
  gulp.src(sources.sass)
  .pipe(sass())
  .pipe(gulp.dest("./public"))
  .on('end', function() {
    cb(null);
  })
  .on('error', function(err) {
    cb(err);
  });
});

3. Return a Promise

gulp.task('sass', function() {
  var deferred = Q.defer();
  gulp.src(sources.sass)
  .pipe(sass())
  .pipe(gulp.dest("./public"))
  .on('end', function() {
    deferred.resolve();
  })
  .on('error', function(err){
    deferred.reject(err);
  });
  return deferred.promise();
});

Remember

It's just Node.
Use normal Javascript functions when you don't need what gulp.task offers.

gulp is:


  • A module with build-centric stream helpers.
  • A module for drop-dead simple CLI.
  • A module to manage build order.

Use gulp for what it does well.

Everything else is just Node.
